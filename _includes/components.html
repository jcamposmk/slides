Componentes

  • Son piezas reutilizables de código
  • Puede contener o no, estado y propiedades
  • Se pueden combinar para construir UI de usuarios complejas
  • Pueden realizar acciones que afecten a otros componentes
  • Se implementan utilizando JSX

Hook de estado

En ocasiones necesitamos guardar datos que cambian durante toda la vida útil de un componente y que causen una renderización de la vista una vez cambiados. Algunos ejemplos:

  • Si se muestran o no todas las características de un plan (vista de Selección de Planes)
  • Si la contraseña es visible o no (vista de Login)
  • El slide activo de un Carousel (vista de Dashboard)
  • Si un KPI se encuentra deshabilitado (vista de Dashboard)
  • Ó simplemente si un botón de encuentra habilitado o no

Para ello se utiliza el hook: useState

Hook de efecto

En ocasiones necesitamos realizar acciones que no forman parte de la responsabilidad principal del componente, sino que son efectos secundarios. Algunos ejemplos:

  • Llamar a una Api
  • Crear temporizadores: setInterval ó setTimeout
  • Cambiar el título del documento con document.title
  • Suscribirse al evento scroll

Para ello se utiliza el hook: useEffect

Hook de efecto

El comportamiento predeterminado es que la función de efecto se ejecute justo después de cada renderizado y al desmontar el componente. Esta ejecución de la función de efecto puede ser condicionado:

  • Especificando una array vacio ([ ]): Desta forma se le indica a React que la función de efecto solo será llamada luego de que el componente sea montado en el DOM
  • Especificando un conjunto de dependencias ([ name, lastName ]): De esta forma, cada vez que cambie name ó lastName React va a llamar a la función de efecto.

Importante: React necesita saber que props o estados se están usando en el efecto, para poder operar con los valores mas recientes. Si se omite alguna prop o estado, puede que dentro del efecto se referencie valores de renderizados anteriores.

Hook de efecto

Hay dos tipos de fectos:

  • Los que no requieren saneamiento:: No necesitan realizar niguna acción para liberar los recursos utilizados. Ejemplo: Manipulación manual del DOM o una llamada a una API
  • Los que si requieren saneamiento: De esta forma, cada vez que cambie name ó lastName React va a llamar a la función de efecto. Ejemplo: Suscripciones a eventos del DOM y temporizadores

Importante: Es necesario liberar los recursos que supongan una carga de memoria para evitar problemas de rendimiento.
